Section 1: What is readable code?

“You know you are working on clean code when each routine you read turns out to be pretty much what you expected.” - Ward Cunningham

Key idea

Code should be written to minimize the time it would take for someone else to understand it.

(that someone might be YOU in a few months!)

Take a typical colleague and measure how much time it took her to read and understand the code.

Understand = she can make changes to it, spot bugs, and understand how it interacts with the rest of the system.

Your job is to minimize this time

Yes, this is hard

  • Getting code to work is the easy part
  • Improving readability requires alot of careful consideration
  • It takes time...but...

If you adopt this goal

  • You will become a better coder
  • You will have fewer bugs
  • You will take more pride in your work
  • You will produce code that everyone around you will love to use.
  • And don't forget...
“The only way to make the deadline — the only way to go fast — is to keep the code as clean as possible at all times.” -Michael Feathers
